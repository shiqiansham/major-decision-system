gi“三重一大”决策与运行管理系统
===============

项目简介
----

“三重一大”决策与运行管理系统是为中国石油大学（北京）克拉玛依校区开发的数字化管理平台，旨在规范“重大决策、重要人事任免、重大项目安排、大额度资金运作”事项的决策流程，实现全过程留痕、闭环管理与透明监督。系统覆盖议题发起、多级审批、会议管理、决策执行、监督回溯等全生命周期，助力高校治理科学化、规范化。

* **委托单位**：中国石油大学（北京）克拉玛依校区

* **承担单位**：短学期23小组

* **开发周期**：2025年6月 – 2025年7月

* **项目状态**：已按时完成开发、测试、试运行并验收

* * *

核心功能模块
------

### 1. 议题管理

* 议题发起与提交（结构化表单、附件上传、自动关联部门）

* 议题分类与优先级（预置5类议题、智能路由）

* 多级审批流程（学院→职能部门→校长办公室三级联审）

* 议题查询与统计分析（多维度筛选、数据看板、效率榜单）

* 个人工作台（待办、已办、历史记录、统计概览）

### 2. 会议管理

* 会议创建与配置（普通/重大会议、强制绑定议题）

* 参会人员与材料管理（动态增删、材料预览/下载）

* 会议执行与控制（状态监控、纪要自动生成、操作留痕）

* 智能会议分析（基于Qwen大模型的偏离度、有序性分析，AI改进建议）

* 会议查询与日历集成（状态筛选、日历视图、统计看板）

### 3. 审批管理

* 审批任务处理（实时通知、待办看板、详情审查）

* 审批决策执行（结构化意见、同意/拒绝/退回、跨部门流转）

* 审批历史追溯（全链记录、时间轴报告、≥5年审计日志）

* 智能决策支持（Qwen大模型分析议题，风险提示）

* 审批数据分析（效能监控、瓶颈识别、一键导出PDF）

### 4. 主页展示系统

* 新闻动态发布（分类浏览、关键词检索、文档预览）

* 决策公开公示（议题决议、会议纪要详情与下载）

* 通知通告管理（定向推送、按角色分发）

* 用户评论互动（反馈收集、热度分析）

### 5. 系统设置与维护

* 权限管理（RBAC四维权限：功能/数据/操作/流程）

* 基础数据维护（决策单位、议题种类、优先级规则灵活配置）

* 系统监控与日志（操作追踪、Prometheus/Grafana性能告警）

* 数据备份与灾备（每日备份、异地存储、4小时内恢复）

* 移动端集成（紧急议题提交、会议接入、人员信息查询）

部署与使用
-----

### 环境要求

* 操作系统：Linux / Windows Server

* Web服务器：Nginx / IIS

* 数据库：MySQL 5.7+

* 运行时：.NET Core 6.0+ / JDK 11+

* 内存：≥8GB（推荐16GB）

* 存储：根据数据量配置，支持对象存储扩展

### 快速开始

1. 克隆仓库：
   bash
   git clone https://github.com/your-repo/sanzhongyida-system.git

2. 导入数据库脚本：
   bash
   mysql -u root -p < database/init.sql

3. 配置后端服务（修改 `appsettings.json` 中的数据库连接、大模型API密钥等）。

4. 启动后端服务：
   bash
   cd backend
   dotnet run
   
   # 或 java -jar model-service.jar

5. 构建前端并部署：
   bash
   cd frontend
   npm install
   npm run build
   
   # 将dist目录复制到web服务器

6. 访问系统：`http://your-server-ip`

详细部署文档请参见 `/docs/deployment.md`。
